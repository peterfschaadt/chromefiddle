What is this and why do I want it?

Each Google Chrome release, Stable, Beta, and Canary, has unique experimental settings that are not enabled by default. Some of these settings control exciting new web features such as real-time WebRTC, SPDY, and Native Client (NaCl). While these cutting-edge features are just appearing in your browser today, many of these features will gain stability and go on to be major features in future Google Chrome releases.

That said, there are reasons that Google hasn't enabled these settings and features by default. Some of these reasons may include performance and stability concerns so be mindful and disable any features that cause issues. The settings you enable in chrome://flags will be active for all Chrome users.

Each operating system, Mac OS X, Windows, Linux, Chrome OS, and Android, all have a different set of experimental settings they can support. Some features may be exclusive to just one operating system.
